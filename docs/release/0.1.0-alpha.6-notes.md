# Circuit 0.1.0-alpha.6 Notes

Date: 2026-05-19

Circuit 0.1.0-alpha.6 is a plugin-only alpha for Claude Code and Codex. The
root `circuit` package remains private in this checkout; npm package publishing
is not part of this alpha.

## Shipping

- Public flow packages: Build, Explore, Fix, Goal, Prototype, Pursue, and
  Review.
- Claude Code: plugin command surfaces for `/circuit:run` and
  `/circuit:handoff`. Built-in flows are routed through Run and remain packaged
  as flow files.
- Codex: bundled Circuit plugin skills for Run and Handoff plus flow files for
  routed execution. Codex host behavior uses the current model-mediated plugin
  surface.
- CLI: `./bin/circuit run --goal "<task>"` and explicit flow invocation,
  including `./bin/circuit run pursue --goal "<task>"`. The experimental
  Create utility remains available as `./bin/circuit create`.

## Fix Lite Exception

Bare `fix:` selects the Fix flow at normal depth. Lite requires an explicit
quick, small, tiny, or simple hint, or an explicit `--rigor lite` flag.

## Proof And Release Checks

The checked-in release proof set is the proof for this alpha. It covers doing
work, deciding, continuity, customization, failure, first run, and plan
execution through [docs/release/proofs/index.yaml](proofs/index.yaml).

Use these release checks for this alpha:

```bash
npm run check-flow-drift
npm run check-release-ready
npm run publish:plugins:check
```

Run `npm run capture-proofs:golden-runs` only when a release diff changes
runtime control flow, flow behavior, command semantics, progress, summaries,
reports, checkpoints, or proof scenarios.

## Non-Shipping In This Alpha

- Native Codex App Server and Claude Agent SDK adapters are not part of this
  alpha scope or the current roadmap.
- A separate Codex worker alias. Codex write-capable worker support uses the
  public `codex` connector.
- Cross-run project-memory query and recall.
- Automatic keep-up-for-you update channel.
- Public `/circuit:pursue` slash command.
- External same-task comparison demo. Current proof is the checked-in release
  proof set, not an external comparison study.

## Host Support Wording

Describe Claude Code and Codex support as plugin command/skill surfaces with
model-mediated host affordances. Do not describe native Codex App Server or
Claude Agent SDK adapters as planned support, and do not claim a polished
generic-shell human progress surface for this alpha.
